Для создания форм можно воспользоваться: 1) конструктором; 2) мастером форм (орешки рекоммендуют).
На первом шаге указываем источник данных. (можно использовать не только таблицу, но и запрос)
Из источника данных выбираем те поля, которые нам необходимо отобразить.
Есть 4 макета. (в нашем примере выберем "в один столбец")
Следующее делаем по умолчанию, потом выбираем любой стиль.
Потом задаем имя формы. (можно оставить по умолчанию, а можно переименовать; можно также уже открыть форму для просмотра)
Завершаем создание формы
Вместо ID пациента можно отображать код пациента.
Ключевые поля - нужно запретить доступ к ним.
Есть два элемента - надпись и поле данных.
Надпись берет значение из названия столбца в структуре. (текст надписи можно изменить(орешки предпочитают изменить на "код пациента"))
На отображение кода слишком большое поле не требуется. (можно также изменить размер (я имею в виду, размер поля) )
Если мы перемещаем элемент, то надпись всегда перемещается вместе с полем данных.
Чтобы выровнять все поля, нужно их все выделить, после чего нажать ПКМ и нажать кнопку, например, "выровнять -> слева".
P.S. Орешки предпочли изменить все имены полей на "Имя", "Фамилия", "Отчество", а также другие поля на полные русские названия...
Чтобы пользователь не менял ключ, нужно закрыть его для доступа. Жмем ПКМ, потом жмем свойства, на вкладке данные (справа откроется большая штука) есть строка "доступ", который по умолчанию открыт, но мы должны его запретить... (поставить вместо "да" -> "нет")
После этого форма будет доступна лишь только для чтения, изменять в ней мы ничего не сможем.
Если требуется более гибкое управление формой, то следует воспользоваться конструктором форм, после чего изменять каждое поле отдельно.
Жмем справа сверху кнопку "добавить поля". Потом перетасикваем все штуки из таблицы pacient (ID, F, I, O, пол, адрес, дата рождения) в конструктор.
Можно разместить любую из этих штук в любом месте формы... "Иногда это удобно..." (с) Орешки
Надпись "Код" и поле "ID пациента" орешки поместили вверху все в одну строку...
Потом орешки сделали строку надписей F, I, O. А под этой строкой 3 поля для вводы данных для каждой из этих надписей.
Все надписи нужно поменять на понятные для пользователя. (Например, F -> Фамилия, N -> Имя, O -> Отчество). Строки для ввода этих данных орешки поместили ровно под каждой из них...
Чтобы красиво и ровно все выровнять, нужно все выделить, нажать ПКМ и "выровнять -> сверху". Это касается как надписей, так и полей для ввода данных.
Надпись пол и поле для выбора пола орешки поместили: ниже трех полей для ввода FIO, справа, в одну строку.
Ниже этого всего орешки в одну строку поместили надпись "дата рождения" и соответствующее поле для ввода данных.
Ниже в одну строку орешки поместили надпись "Адрес" и большоооое поле для ввода данных под него (прямоугольник по длине и ширине в 3-4 раза больше, чем поле надписи "Адрес").
В конструкторе можно выбрать цветовую схему (сверху слева квадратная кнопка).
После всего этого в получившейся форме регистратору (работающем с нашей формой) можно изменять все, кроме кода пациента и пола (так как они указываются однозначно сразу)...
Поскольку все окно формы представляет собой одну запись, то перемещение по ним возможно лишь по кнопочкам внизу (влево, вправо; есть также поле, куда можно ввести данные о желаемой форме)
Лучше всего стараться всю информацию уложить на один экран, чтобы пользователю не пришлось листать вверх-вниз (в крайнем случае использовать вкладки).
Предположим, нам также нужно узнать какие визиты были у того или иного пациента... У нас есть табличка визит, где написано, какие визиты были у пациента... 
Тут нужно воспользоваться составной формой... Для начала нужно сделать основную форму, где отображаются сведения из главной таблицы, а потом сделать подчиненную форму, где сведения отображаются из таблицы, связанной с главной.
Для этого мы продолжаем работать с конструктором.
Сразу указываем в качестве источника данных табличку patient.
Теперь на конструктор формы мы перемещаем данные таблицы, связанной с главной. Справа сверху ищем маленькую кнопку "подчиненные формы" (или как-то так). Открывается окно... В него указываем нашу дополнительную таблицу visit.
Перетаскиваем из таблички visit следующие поля: ID_visit (Орешки переименовали надпись в "№" и сразу запретили доступ к измененю этого поля), дата визита, цель визита.
Перемещенную дополнительную таблицу visit (ее поля) помещаем в конструктор под основной таблицей patient (под ее полями)... Но на подчиненную форму нужно также добавить поле внешнего ключа!!! Поэтому добавляем ID_patient в подчиненную форму, но его можно и нужно не отображать. Оно используется только для соединения строк.
Далее, в мастере подчиненных форм нужно выбрать самостоятельное подчинение, где нужно связать правильно ID пациента и ID визита в главной и дополнительной таблицах...
Если подчиненную форму сделать в виде таблицы, то надобность переходов почти отпадает (с) орешки.
Еще один момент - это использование элементов управления... Можно создать еще один элемент управления специально для перелистывания... Access предоставляет шаблон кнопок...
